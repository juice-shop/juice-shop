# Checklist: OWASP Juice Shop Release Notes

- [ ] **Disclaimer**: Does the release include `⚡`, `⚠️`, or `📜` changes? If so, is there a disclaimer blockquote at the top?
- [ ] **Challenges**: Are all new challenges listed with their difficulty rating (e.g., `⭐⭐-challenge`)?
- [ ] **Breaking Challenges**: Do breaking challenge changes have the `⚡` icon?
- [ ] **Technical Breaking**: Are technical breaking changes (Node.js version, removals) marked with `⚠️`?
- [ ] **Kudos**: Are external contributors properly credited with `(kudos to @username)`?
- [ ] **Iconography**: Do all section headings have their corresponding emoji?
- [ ] **References**: Are PRs and commits referenced using `#number` and `hash` format where appropriate?
- [ ] **Categorization**: Is the categorization accurate (Runtime, Frontend, Challenges, UI, etc.)?
- [ ] **Language**: Are translation expanded for specific languages mentioned in the `🌐 I18N` section?
